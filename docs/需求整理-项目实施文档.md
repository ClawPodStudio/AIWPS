## 一、项目概述

### 1.1 项目背景

**目标用户**：初高中学生（单科个性化辅导）

**核心场景**：
1. 上传各省中高考题库 → 整理知识点 → 学生刷题
2. 错题整理 → 知识点归纳 → AI生成变形题 → 针对性提高
3. 艾宾浩斯记忆曲线 → 智能复习计划

### 1.2 系统角色

| 角色 | 描述 | 核心功能 |
|------|------|---------|
| 学生 | 学习者 | 刷题、错题本、AI推荐练习 |
| 老师 | 教学者 | 布置作业、生成试卷、查看学生情况 |
| 机构/学校 | 管理方 | 查看整体数据、学生提升情况 |
| 管理员 | 系统管理 | 租户管理、系统配置 |

### 1.3 多租户架构

- ✅ 多租户隔离（租户 = 机构/教育局）
- ✅ 学生绑定到不同的学校/老师
- ✅ 老师可以管理多个班级

## 三、待开发功能（需求遗漏清单）

- 前端：知识点管理、题目绑定、筛选

---

### 3.2 错题智能排序 🔴 高优先级

**需求描述**：
> 错题可以根据历年题库中常出现的考点和得分容易程度进行排序，优先对高得分点、低难度的题进行排序

**现状**：错题列表无排序逻辑

**待实现**：
1. **考点频次**：统计知识点在历年真题中出现的次数
2. **得分容易程度**：可配置知识点得分权重
3. **难度系数**：低难度题目优先
4. **综合排序算法**：考点频次 × 得分权重 × (1/难度)

---

### 3.3 AI生成变形题 🔴 高优先级

**需求描述**：
> 根据错题，将知识点重新归纳，再出一份该知识点的相应变形题，以此来提高这类题型的正确率

**现状**：无AI出题能力

**待实现**：
1. 知识点 → 变形题生成
2. 题目微调（改数值、改问法、保留核心考点）
3. 难度可控（保持/提高/降低）
4. 题目审核机制（AI生成 → 老师审核 → 学生可见）

---

### 3.4 艾宾浩斯记忆曲线出题 📌 中优先级

### 3.10 题库批量上传与智能解析 🟡 低优先级

**需求描述**：
> 我需要上传某省所有年份的所有的中高考题库，题库上传需要支持OCR，上传pdf，上传word

**现状**：手动单条录入

**待实现**：

#### 3.10.1 支持的输入格式
| 格式 | 说明 | 处理方式 |
|------|------|---------|
| PDF | 历年真题PDF | PDF解析 + OCR识别 |
| Word(.docx) | 历年真题文档 | Word解析 |
| Excel | 结构化题库导入 | 模板导入 |
| 图片 | 试卷拍照 | OCR识别 |
| 文本 | 纯文本题目 | 正则解析 |

#### 3.10.2 OCR识别能力
- 印刷体识别（高精度）
- 手写体识别（低精度，提示人工确认）
- 公式识别（数学公式、化学式）
- 几何图形坐标提取

#### 3.10.3 智能解析流程
```
上传文件 → 格式检测 → 文本提取 → 题目拆分 → 题型判断 → 知识点关联 → 难度评估 → 预览确认 → 正式入库
```

#### 3.10.4 题目自动拆分规则
- 按题号拆分（1.、2.、(1)、(2)）
- 选择题：识别选项A/B/C/D
- 填空题：识别下划线___
- 解答题：识别"解"、"证明"、"计算"
- 作文题：识别字数要求

#### 3.10.5 知识点自动关联
- AI分析题目文本
- 匹配已有知识点库
- 未匹配知识点 → 提示新建

#### 3.10.6 质量校验
- 答案完整性检查
- 知识点覆盖率统计
- 重复题目检测
- 人工抽检确认

#### 3.10.7 技术方案
| 能力 | 技术选型 |
|------|---------|
| PDF解析 | Apache PDFBox / pdfplumber |
| Word解析 | Apache POI |
| OCR识别 | Tesseract（开源）或 百度OCR API（商用） |
| 公式识别 | Mathpix / LaTeX-OCR |
| AI解析 | Spring AI + 大模型 |

## 四、数据库变更清单

### 4.1 需新增表

```sql
-- 知识点题型关联
CREATE TABLE knowledge_point_type (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    knowledge_point_id BIGINT NOT NULL,
    question_type VARCHAR(20) NOT NULL COMMENT 'CHOICE/FILL/BLANK/DRAWING/ESSAY',
    weight DECIMAL(5,2) DEFAULT 1.0 COMMENT '题型权重',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_kp (knowledge_point_id),
    INDEX idx_type (question_type)
);

-- 知识点掌握度
CREATE TABLE knowledge_mastery (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    student_id BIGINT NOT NULL,
    knowledge_point_id BIGINT NOT NULL,
    mastery_level DECIMAL(5,2) DEFAULT 0 COMMENT '掌握度0-100',
    correct_count INT DEFAULT 0,
    wrong_count INT DEFAULT 0,
    last_review_time DATETIME,
    next_review_time DATETIME COMMENT '艾宾浩斯下次复习时间',
    is_removed TINYINT DEFAULT 0 COMMENT '是否已从出题范围移除',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_student_kp (student_id, knowledge_point_id)
);

-- 变形题记录
CREATE TABLE variant_question (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    original_question_id BIGINT NOT NULL,
    variant_question_id BIGINT NOT NULL,
    variant_type VARCHAR(20) COMMENT 'NUMBER/PHRASE/CONTEXT...',
    generated_by VARCHAR(20) COMMENT 'AI/MANUAL',
    status TINYINT DEFAULT 0 COMMENT '0待审核 1已通过',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 试卷模板
CREATE TABLE paper_template (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    tenant_id BIGINT NOT NULL,
    name VARCHAR(100) NOT NULL,
    subject_id BIGINT NOT NULL,
    grade_id BIGINT,
    question_type_config JSON NOT NULL COMMENT '{"CHOICE":10,"BLANK":5}',
    difficulty_config JSON COMMENT '{"easy":0.3,"medium":0.5,"hard":0.2}',
    total_score INT DEFAULT 100,
    time_limit INT COMMENT '分钟',
    created_by BIGINT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 知识点考点频次
CREATE TABLE knowledge_exam_freq (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    knowledge_point_id BIGINT NOT NULL,
    province VARCHAR(50),
    year INT,
    exam_type VARCHAR(20) COMMENT '中考/高考',
    frequency INT DEFAULT 1 COMMENT '出现次数',
    score_weight DECIMAL(5,2) DEFAULT 1.0 COMMENT '得分权重',
    INDEX idx_kp (knowledge_point_id),
    INDEX idx_province_year (province, year)
);
```

## 七、技术实现建议

### 7.1 AI出题方案

**方案A：RAG + 大模型**
```
题库 → 向量化 → 向量库
                ↓
用户请求 → 检索相关题目 → 构建Prompt → 大模型生成 → 返回
```

**方案B：题目微调模型**
```
收集高质量题目 → 微调小模型 → 生成变形题
```

**推荐**：方案A，实现成本低，效果足够

### 7.2 艾宾浩斯实现

```java
// 记忆周期（天）
int[] intervals = {1, 2, 4, 7, 15, 30};

// 下次复习时间 = 当前时间 + 间隔天数 × 遗忘曲线因子
double factor = Math.pow(0.9, wrongCount); // 错题越多，间隔越短
int nextInterval = (int)(intervals[reviewCount] * factor);
```
